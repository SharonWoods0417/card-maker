## 📂 **目录详细说明**

### 📚 **doc/ 目录 - 项目文档**
**作用**：存放所有项目相关的文档和说明
- **重要性**：⭐⭐⭐⭐⭐ 非常重要，记录项目需求和技术方案
- **维护频率**：经常更新
- **初学者说明**：这里存放的是"纸质文档"，帮助理解项目需求和技术实现
- **包含文档**：
  - **00_文档导读** ⭐⭐⭐⭐⭐ 项目导航手册，新chat必读
  - **01_需求规格说明** ⭐⭐⭐⭐⭐ 项目要做什么
  - **02_技术分析与架构设计** ⭐⭐⭐⭐⭐ 技术方案和架构
  - **03_项目目录结构说明** ⭐⭐⭐⭐ 代码结构导航（本文档）
  - **04_任务执行记录** ⭐⭐⭐⭐⭐ 开发进度管理
  - **05_问题记录工具** ⭐⭐⭐ 问题解决经验

### 💻 **src/ 目录 - 源代码**
**作用**：存放所有的程序源代码
- **重要性**：⭐⭐⭐⭐⭐ 最重要，这是程序的核心
- **维护频率**：每天都会修改
- **初学者说明**：这里是"写代码的地方"，所有功能都在这里实现

#### 🔌 **src/api/ - API服务模块（已完成）**
**作用**：封装所有外部API调用，如OpenAI、Pexels图片、词典API
- **重要性**：⭐⭐⭐⭐⭐ 核心功能，连接外部服务
- **维护频率**：开发期间经常修改
- **初学者说明**：这是"对外联络部门"，负责安全地调用外部API获取数据
- **包含文件**：
  - **types.ts** ⭐⭐⭐⭐⭐ 定义API数据格式，确保类型安全 ✅
  - **openai.ts** ⭐⭐⭐⭐⭐ OpenAI智能补全服务 ✅ 已完成
  - **image.ts** ⭐⭐⭐⭐ Pexels图片搜索服务 ✅ 已完成
  - **dictionary.ts** ⭐⭐⭐ 词典API备用服务 ✅ 已完成

#### ⚛️ **src/components/ - React组件**
**作用**：存放React组件文件，每个组件负责一个功能模块
- **重要性**：⭐⭐⭐⭐⭐ 核心功能实现
- **初学者说明**：把网页拆分成小块，每个文件负责一个小块的功能

#### 📝 **src/types/ - 类型定义**
**作用**：定义TypeScript数据类型，让代码更安全
- **重要性**：⭐⭐⭐⭐ 代码质量保证
- **初学者说明**：告诉程序"数据应该长什么样子"，防止数据类型错误

#### 🔧 **src/utils/ - 工具函数**
**作用**：存放可复用的工具函数
- **重要性**：⭐⭐⭐ 代码复用和整洁
- **初学者说明**：把常用的小功能单独提取出来，避免重复写代码

### 📦 **node_modules/ 目录 - 依赖包**
**作用**：存放项目依赖的第三方库
- **重要性**：⭐⭐⭐⭐⭐ 必须存在，但不要手动修改
- **大小**：通常很大（几百MB）
- **初学者说明**：这是"别人写好的代码库"，npm自动管理，不要动它！

### ⚡ **.bolt/ 目录 - Bolt工具配置**
**作用**：Bolt AI工具的配置文件
- **重要性**：⭐⭐ 工具相关，非核心功能
- **初学者说明**：如果不用Bolt工具了，可以删除这个目录

### 🆚 **.vscode/ 目录 - VS Code配置**
**作用**：VS Code编辑器的项目配置
- **重要性**：⭐ 编辑器相关，可选
- **初学者说明**：因为用的是Cursor不是VS Code，这个目录基本用不到

### 🌐 **public/ 目录 - 静态资源**
**作用**：存放网站的静态文件，可直接通过URL访问
- **重要性**：⭐⭐⭐⭐ 重要，存放字典文件等资源
- **维护频率**：偶尔更新
- **初学者说明**：这里的文件可以直接在网页上访问，比如图片、JSON文件等

#### 📚 **public/dictionaries/ - 字典文件目录**
**作用**：存放本地词典JSON文件
- **重要性**：⭐⭐⭐⭐⭐ 核心功能，离线词典系统
- **维护频率**：系统运行时自动更新
- **初学者说明**：这是"单词字典库"，存放已知单词的释义、音标等信息
- **包含文件**：
  - **base.json** ⭐⭐⭐⭐⭐ 系统基础词典（预制词汇）
  - **custom.json** ⭐⭐⭐⭐ 用户自定义词典（AI补全结果）

### 🎯 **src/services/ - 业务服务模块**
**作用**：存放业务逻辑服务代码，连接API和本地存储
- **重要性**：⭐⭐⭐⭐⭐ 核心功能，业务逻辑处理
- **维护频率**：开发期间经常修改
- **初学者说明**：这是"业务处理中心"，负责具体的业务逻辑处理
- **包含文件**：
  - **dictionary.ts** ⭐⭐⭐⭐⭐ 字典查询服务（已完成）

## 📄 **重要文件详细说明**

### 📋 **配置文件（项目运行必需）**

#### **package.json** ⭐⭐⭐⭐⭐
**作用**：项目的"身份证"，声明项目信息和依赖
```json
{
  "name": "项目名称",
  "dependencies": "项目需要哪些第三方库",
  "scripts": "可以运行哪些命令"
}
```
**初学者说明**：告诉npm"这个项目需要什么"，绝对不能删除

#### **package-lock.json** ⭐⭐⭐⭐⭐
**作用**：锁定依赖包的具体版本
**初学者说明**：确保所有人安装的依赖版本完全一样，不要手动修改

#### **vite.config.ts** ⭐⭐⭐⭐
**作用**：Vite构建工具的配置文件
**初学者说明**：告诉Vite"怎么把代码打包成网页"，包括端口设置等

#### **tsconfig*.json** ⭐⭐⭐⭐
**作用**：TypeScript编译器配置
**初学者说明**：告诉TypeScript"怎么检查代码和转换成JavaScript"

#### **tailwind.config.js** ⭐⭐⭐
**作用**：TailwindCSS样式框架配置
**初学者说明**：设置CSS样式的规则和主题

#### **.env.local.example** ⭐⭐⭐⭐⭐
**作用**：示例环境变量文件，避免泄露真实 API Key
**初学者说明**：复制为 `.env.local` 并填写您的 OpenAI / Pexels KEY，文件已被 .gitignore 排除，不会上传到 GitHub

#### **eslint.config.js** ⭐⭐⭐
**作用**：代码质量检查工具配置
**初学者说明**：自动检查代码是否规范，提醒代码问题

### 🌐 **入口文件**

#### **index.html** ⭐⭐⭐⭐⭐
**作用**：网页的HTML入口文件
**初学者说明**：浏览器访问时最先加载的文件，连接React应用

#### **src/main.tsx** ⭐⭐⭐⭐⭐
**作用**：React应用的JavaScript入口
**初学者说明**：告诉React"从这里开始运行"

#### **src/App.tsx** ⭐⭐⭐⭐⭐
**作用**：主应用组件，整个应用的根组件
**初学者说明**：整个网页应用的"大boss"，其他组件都是它的"手下"

### 🎨 **样式文件**

#### **src/index.css** ⭐⭐⭐⭐
**作用**：全局CSS样式文件
**初学者说明**：设置整个网站的样式和外观

### 📖 **文档文件**

#### **README.md** ⭐⭐⭐
**作用**：项目说明文档，GitHub首页显示
**初学者说明**：项目的"说明书"，告诉别人这个项目是干什么的

#### **LICENSE** ⭐⭐
**作用**：开源协议文件
**初学者说明**：规定别人可以怎么使用你的代码

### 🚫 **.gitignore** ⭐⭐⭐⭐

## 🔧 **开发环境说明**

### **本地开发服务器**
- **启动命令**：`npm run dev`

### **代码检查**
- **运行命令**：`npm run lint`
- **配置文件**：eslint.config.js

### **构建打包**
- **运行命令**：`npm run build`
- **输出目录**：dist/ （构建时自动生成）

## 📝 **维护建议**

### **经常更新的文件**
- `src/` 目录下的所有文件（开发时）
- `doc/` 目录下的文档（需求变更时）

## 🎯 **初学者学习路径建议**

1. **先理解项目结构**：从最重要的文件开始 → package.json → src/App.tsx
2. **学习组件结构**：src/components/ 目录，理解每个组件的功能
3. **理解数据流**：src/types/ 和 src/utils/，学习数据如何定义和处理
4. **掌握配置文件**：逐步理解各种 .config 文件的作用
5. **学习构建流程**：理解从源代码到最终网页的转换过程
